
# 🎨 Сравнение раннего и позднего стиля иллюстраций Ивана Билибина

Автоматический анализ иллюстраций Ивана Билибина с использованием **GPT-4o** и классических алгоритмов компьютерного зрения.  
Проект позволяет исследовать, как менялся стиль художника на протяжении двух периодов его творчества.

## 📁 Структура репозитория

```
bilibin-art-analyzer/
├── bilibin_analysis.py         # основной скрипт: анализ изображений и генерация CSV-датасета
├── requirements.txt            # зависимости Python
├── .env.example                # образец файла с API-ключом
├── .gitignore
├── early_images/               # изображения раннего периода (загружаются локально)
├── late_images/                # изображения позднего периода (загружаются локально)
├── bilibin_analysis.csv        # итоговый датасет с результатами анализа
└── README.md
```

## ✨ Что делает проект

Для каждого изображения вычисляются:

- **Жанр**, **основные объекты**, **эмоциональная окраска** (через GPT-4o)
- **Динамика/статика** (–1…+1), **тип перспективы**, **глубина сцены**, **композиционная сложность**, **живописность** (через GPT-4o)
- **Палитра** – 5 доминирующих цветов (алгоритмически, k‑means)
- **Тональность** – средняя яркость, нормализованная от –1 (тёмное) до +1 (светлое)
- **Цветовая гармония** – тип схемы (монохромная, аналогичная, комплементарная, триадная)

На основе этих данных строятся сравнительные графики и выполняются статистические тесты для выявления изменений в творчестве Билибина.

## 🚀 Быстрый старт

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/yourusername/bilibin-art-analyzer.git
cd bilibin-art-analyzer
```

### 2. Установите зависимости

```bash
pip install -r requirements.txt
```

### 3. Настройте API-ключ OpenRouter

- Зарегистрируйтесь на [openrouter.ai](https://openrouter.ai/) и получите ключ.
- Создайте файл `.env` в корне проекта со строкой:

```env
OPENROUTER_API_KEY=sk-or-v1-ваш_ключ
```

> Можно также экспортировать переменную окружения вручную:
> ```bash
> export OPENROUTER_API_KEY="sk-or-v1-..."
> ```

### 4. Добавьте изображения

Разместите иллюстрации в папках `early_images/` (ранние работы) и `late_images/` (поздние работы).  
**Форматы:** `.png`, `.jpg`, `.jpeg`.  
Файлы будут обработаны в алфавитном порядке (первые N из каждой папки, настраивается в конфиге).

### 5. Запустите анализ

```bash
python bilibin_analysis.py
```

Прогресс сохраняется после каждого изображения. При прерывании скрипт можно перезапустить – он автоматически продолжит с того места, где остановился.

Результат будет сохранён в `bilibin_analysis.csv`.

### 6. Визуализируйте данные

```bash
python visualise_bilibin.py
```

Появятся графики и статистические выводы в консоли, а также PNG-файлы с визуализациями.

## ⚙️ Конфигурация

Основные параметры задаются в словаре `CONFIG` внутри `bilibin_analysis.py`:

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `model_name` | Модель OpenRouter | `openai/gpt-4o` |
| `max_images_early` | Макс. количество ранних работ | 40 |
| `max_images_late` | Макс. количество поздних работ | 28 |
| `image_max_size` | Макс. размер изображения в пикселях | 768 |
| `image_quality` | Качество JPEG (1‑100) | 70 |
| `delay_between_requests` | Пауза между запросами, сек. | 0.5 |
| `resume` | Автоматически продолжать с места остановки | `True` |

## 📈 Оценка расхода токенов

При сжатии до 768×768 и качестве 70% одно изображение расходует примерно **85 токенов** на кодирование.  
С учётом короткого промпта и ответа **общий расход на одну картину составляет около 435 токенов**.  
Таким образом, 68 изображений потребуют примерно **29 580 токенов** – это укладывается в стандартный бесплатный лимит многих API-провайдеров.
